The rapid growth of digital payment systems and Unified Payments Interface (UPI) transactions has significantly increased the risk of fraudulent activities in online financial systems. Fraudulent transactions can lead to financial losses, security issues, and customer distrust.
This project focuses on analyzing UPI transaction data to identify fraudulent patterns and risky transaction behavior using SQL, Power BI, and Python.
A synthetic dataset containing around 8500 transaction records was generated to simulate real-world UPI transactions. SQL was used for data cleaning, preprocessing, and analysis. Power BI was used to create interactive dashboards and KPIs for fraud monitoring, while Python was used for data visualization and exploratory data analysis.
The project identifies fraud patterns using behavioral indicators such as failed transaction attempts, new device usage, night transactions, and transaction velocity. The dashboards and visualizations provide insights into high-risk transaction behavior and fraud trends.


Problem Statement
Digital payment systems generate a huge number of transactions every day. Detecting fraudulent activities among these transactions is difficult using traditional monitoring methods.
The problem addressed in this project is:
“To analyze UPI transaction data and identify potentially fraudulent transactions using data cleaning, dashboard analysis, and visualization techniques.”

Business Insights
The following insights were derived from the project:
•	Transactions from new devices have higher fraud probability
•	High failed attempts are strong indicators of suspicious activity
•	Night transactions show increased fraud behavior
•	High transaction velocity may indicate fraudulent activity
•	Certain locations and transaction types are more vulnerable to fraud
•	Behavioral indicators are more effective than transaction amount alone
